# REQUIREMENTS: NSFW Image Generation Service

## Functional Requirements

### FR-001: 프롬프트 기반 이미지 생성
- 사용자가 텍스트 프롬프트를 입력하면 이미지를 생성한다
- 네거티브 프롬프트를 지원한다
- 한국어/영어 프롬프트를 모두 지원한다

### FR-002: 스타일 참조 이미지
- 사용자가 스타일 참조 이미지를 업로드할 수 있다
- 업로드된 이미지의 화풍/스타일을 생성 이미지에 반영한다
- IP-Adapter를 활용하여 스타일을 전이한다

### FR-003: 캐릭터 참조 이미지
- 사용자가 캐릭터 참조 이미지를 업로드할 수 있다
- 업로드된 캐릭터의 외모/특징을 생성 이미지에 반영한다
- IP-Adapter FaceID를 활용하여 캐릭터 일관성을 유지한다

### FR-004: 생성 파라미터 조정
- Inference Steps (1-100, default: 30)
- CFG Scale (1-20, default: 7.5)
- Image Size (512x512 ~ 1024x1024)
- Seed (random or fixed)
- LoRA 선택 및 가중치 조정

### FR-005: LoRA 지원
- HuggingFace Hub의 LoRA 모델을 경로로 지정할 수 있다
- LoRA 가중치(scale)를 0.0-1.0 범위로 조정할 수 있다
- 다중 LoRA 동시 적용을 지원한다

### FR-006: 갤러리 및 스크랩
- 생성된 이미지를 그리드 형태로 표시한다
- 이미지 클릭 시 원본 크기로 표시 + 메타데이터 표시
- 이전 생성의 파라미터를 복사하여 재생성할 수 있다
- 이미지 다운로드 및 삭제가 가능하다

### FR-007: 비동기 생성 + 진행 상태
- 이미지 생성은 비동기로 처리한다
- 생성 중 로딩/진행 상태를 표시한다
- 생성 완료 시 자동으로 결과를 표시한다

## Non-Functional Requirements

### NFR-001: 성능
- 이미지 1장 생성: 30초 이내 (SDXL 30 steps 기준)
- 페이지 로드: 2초 이내

### NFR-002: 사용 환경
- 개인 사용 목적 (localhost)
- 인증 불필요
- 데스크톱 브라우저 최적화

### NFR-003: 비용
- fal.ai pay-per-use 모델
- 이미지당 예상 비용: $0.01 ~ $0.05

### NFR-004: 저장
- 생성 이미지: 로컬 파일시스템 (output/ 디렉토리)
- 메타데이터: JSON 파일 (이미지와 함께 저장)

## Acceptance Criteria

### AC-001: 기본 생성
- [x] 프롬프트 입력 → Generate 클릭 → 이미지 생성 완료 → 화면에 표시

### AC-002: 스타일 참조
- [x] 스타일 이미지 업로드 → 생성된 이미지가 해당 스타일을 반영

### AC-003: 캐릭터 참조
- [x] 캐릭터 이미지 업로드 → 생성된 이미지에 해당 캐릭터의 특징이 반영

### AC-004: LoRA 적용
- [x] LoRA 경로 입력 + 가중치 설정 → 생성 이미지에 LoRA 스타일 반영

### AC-005: 갤러리
- [x] 생성된 이미지들이 갤러리에 표시되고, 메타데이터 확인 및 다운로드 가능
